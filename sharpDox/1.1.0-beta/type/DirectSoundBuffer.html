Lock(Int32 offset, Int32 bytes, IntPtr& audioPtr1, Int32& audioBytes1, IntPtr& audioPtr2, Int32& audioBytes2, DSBLock lockFlags)

Readies all or part of the buffer for a data write and returns pointers to which data can be written.

public void Lock(Int32 offset, Int32 bytes, IntPtr& audioPtr1, Int32& audioBytes1, IntPtr& audioPtr2, Int32& audioBytes2, DSBLock lockFlags)

Parameters

offset

Int32

Offset, in bytes, from the start of the buffer to the point where the lock begins. This parameter is ignored if FromWriteCursor is specified in the lockFlags parameter.

bytes

Int32

Size, in bytes, of the portion of the buffer to lock. The buffer is conceptually circular, so this number can exceed the number of bytes between offset and the end of the buffer.

audioPtr1

IntPtr&

Receives a pointer to the first locked part of the buffer.

audioBytes1

Int32&

Receives the number of bytes in the block at audioPtr1. If this value is less than bytes, the lock has wrapped and audioPtr2 points to a second block of data at the beginning of the buffer.

audioPtr2

IntPtr&

Receives a pointer to the second locked part of the capture buffer. If Zero is returned, the audioPtr1 parameter points to the entire locked portion of the capture buffer.

audioBytes2

Int32&

Receives the number of bytes in the block at audioPtr2. If audioPtr2 is Zero, this value is zero.

lockFlags

DSBLock

Flags modifying the lock event.

SetFrequency(Int32 frequency)

Sets the frequency at which the audio samples are played.

public void SetFrequency(Int32 frequency)

Parameters

frequency

Int32

Frequency, in hertz (Hz), at which to play the audio samples. A value of FrequencyOriginal resets the frequency to the default value of the buffer format.

Remarks

Before setting the frequency, you should ascertain whether the frequency is supported by checking the MinSecondarySampleRate and MaxSecondarySampleRate members of the DirectSoundCapabilities structure for the device. Some operating systems do not support frequencies greater than 100,000 Hz.
