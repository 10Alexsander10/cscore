Looping

After the end of the audio buffer is reached, play restarts at the beginning of the buffer. Play continues until explicitly stopped. This flag must be set when playing a primary buffer.

TerminateByDistance

If the hardware has no available voices, a currently playing buffer will be stopped to make room for the new buffer. The buffer prematurely terminated will be selected from buffers that have the buffer's Mute3DAtMaxDistance flag set and are beyond their maximum distance. If there are no such buffers, the method fails.

public const DSBPlayFlags TerminateByDistance

Constant Value

16

TerminateByPriority

If the hardware has no available voices, a currently playing buffer will be stopped to make room for the new buffer. The buffer prematurely terminated will be the one with the lowest priority as set by the priority parameter passed to Play for the buffer.

public const DSBPlayFlags TerminateByPriority

Constant Value

32

TerminateByTime

If the hardware has no available voices, a currently playing nonlooping buffer will be stopped to make room for the new buffer. The buffer prematurely terminated is the one with the least time left to play.

public const DSBPlayFlags TerminateByTime
